# Vanish-py — Seam Carving Design

**Date:** 2026-07-10
**Status:** Approved (design phase)

## Summary

Vanish-py is a content-aware image resizing tool built on **seam carving**. Instead of
uniformly squishing or cropping, it removes the least-important winding pixel paths
("seams") through an image, so important subjects (people, objects) stay undistorted
while low-detail background is absorbed. It ships three features on a shared algorithm
core:

1. **Shrinking** — content-aware width/height reduction.
2. **Enlarging** — seam insertion (the same math in reverse).
3. **Object removal** — mask a subject and route seams through it so it vanishes.

Delivered as an importable Python library **and** a thin CLI (`python -m vanish ...`),
with a fully vectorized NumPy pipeline and a naive-loop reference kept only to produce a
benchmark speedup number.

## Goals

- A pure, testable NumPy algorithm core (the DP is the intellectual centerpiece).
- Three self-contained features composed from that core.
- Both a clean library API and a thin CLI.
- A vectorized pipeline plus a naive-vs-vectorized benchmark.
- Catchy, reproducible README demos generated by scripts.

## Non-Goals (YAGNI)

- GUI / interactive mask painting.
- Forward-energy (Rubinstein) seam variant.
- Video seam carving.
- GPU acceleration.
- Local energy re-computation optimization (recompute full map each iteration).

Each is noted in the README as a possible future extension.

## Decisions (from brainstorming)

| Decision | Choice |
|----------|--------|
| Feature scope | All three: shrink + enlarge + object removal |
| Interface | CLI + importable library |
| Performance | Fully vectorized NumPy + naive reference for a benchmark |
| Object-removal mask input | Mask PNG (white = remove); no GUI |
| Demo assets | All four: hero GIF, side-by-side vs dumb resize, object-removal before/after, seam overlay |
| Code structure | Approach A: functional core + feature layer + thin CLI, with a thin stateful `Carver` driver |
| Removal loop | Recompute full energy map + DP after each seam removal (simplest, correct) |
| Horizontal seams | Transpose the array and reuse the vertical-seam code |

## Architecture

Approach A — a pure-functions core operating on NumPy arrays, a feature layer that
composes it, an I/O layer isolating Pillow, and a thin CLI. The only stateful piece is a
small `Carver` driver that streams frames, which the GIF recorder and seam-overlay demo
reuse.

### Package layout


### Layer responsibilities & dependencies

- **`core.py`** — the intellectual core. Functions on `np.ndarray` only. Depends on
  NumPy alone. Knows nothing about files, masks, or CLI.
- **`carver.py`** — a thin `Carver` holding the working image; drives
  "find → remove → repeat" and exposes a generator yielding `(image, seam)` per step.
  The only stateful piece; what the GIF recorder and seam-overlay iterate over.
- **`features.py`** — user-facing operations: `resize(img, width=, height=)`,
  `enlarge(...)`, `remove_object(img, mask)`. Compose core + carver; handle transpose
  for horizontal seams.
- **`io.py`** — Pillow load/save; mask PNG → boolean array. Isolates the third-party
  I/O dependency; normalizes grayscale/RGBA inputs to RGB so `core` only sees `(H,W,3)`.
- **`cli.py`** — parses args, calls `io` + `features`, writes output. No algorithm logic.
- **`_naive.py`** — quarantined reference implementation; imported only by `benchmark.py`.

**Dependencies:** `numpy` (core), `pillow` (I/O), `imageio` (GIF export, examples only).
Sobel is hand-rolled with NumPy slicing — no SciPy — to keep the vectorization story
self-contained.

## Data flow (shrink example)

```
CLI args ──▶ io.load_image ──▶ ndarray(H,W,3)
                                   │
        features.resize(width=W-N) │
                                   ▼
        Carver: loop N times ── core.energy_map ─▶ core.cumulative_energy
                                   │                        │
                                   ▼                        ▼
                          core.remove_seam ◀── core.find_vertical_seam
                                   │
                                   ▼
                        ndarray(H, W-N, 3) ──▶ io.save_image
```

- **Height changes:** `features` transposes the array, runs the identical vertical-seam
  path, transposes back.
- **Object removal:** `remove_object` biases the energy map to −∞ under the mask so
  seams route *through* the target, removing seams until the masked region is gone.
  Optional `shrink=True` leaves the image narrower; otherwise the removed columns are
  re-inserted elsewhere via enlargement to restore original width.

## Algorithm core (`core.py`)

### `energy_map(img) -> float64[H,W]`
Convert to grayscale (luminance-weighted). Compute Sobel gradients in x and y via NumPy
slicing / `np.pad` (no Python pixel loop), combine as `sqrt(gx² + gy²)`. Edges/faces
score high; flat sky/sand low.

### `cumulative_energy(energy) -> (M[H,W], backtrack[H,W])`
The DP. Row 0 = energy row 0. For each subsequent row:

```
M[r,c] = energy[r,c] + min(M[r-1, c-1], M[r-1, c], M[r-1, c+1])
```

Column edges are handled so out-of-range neighbors do not wrap. **Vectorized across
columns within the row** (build the three shifted candidate arrays, take element-wise
min); the loop over *rows* stays sequential because each row depends on the one above —
the honest limit of vectorization here, and exactly the recurrence an interviewer probes.
`backtrack[r,c]` records which of the three parents was chosen.

### `find_vertical_seam(energy) -> int[H]`
`argmin` of `M`'s bottom row → seam end; follow `backtrack` upward to recover one column
index per row.

### `remove_seam(img, seam) -> img[H, W-1]` and `insert_seam(img, seam)`
Delete / duplicate one pixel per row (duplicate = average of the seam pixel and a
neighbor, for smooth enlargement). Boolean-mask + reshape, vectorized.

### Enlargement caveat (baked in)
To add N columns, find the N lowest-energy seams *on the original* first (removing them
virtually so the same seam is not picked N times), then insert them together. Inserting
one-at-a-time stacks duplicates on the single cheapest seam and creates a smear — a known
trap, documented.

## CLI (`python -m vanish ...`)

```
vanish shrink  IN OUT (--width W | --dw N) [--height H | --dh N]
vanish enlarge IN OUT (--width W | --dw N) [--height H | --dh N]
vanish remove  IN OUT --mask MASK.png [--shrink]   # --shrink also narrows to fill the gap
vanish energy  IN OUT                               # dump the energy map (README-friendly)
vanish seams   IN OUT --count K                     # overlay K seams in red
```

Library mirror: `vanish.resize(img, width=, height=)`, `vanish.enlarge(...)`,
`vanish.remove_object(img, mask, shrink=False)`.

## Error handling

- **Validation at the boundary** (`io`/`cli`): missing file, unreadable image, mask
  shape ≠ image shape, target dimension ≤ 0 or ≥ current (for shrink) → clear
  `ValueError` / `SystemExit` with a readable message, not a traceback.
- **Core stays trusting**: pure functions assume validated input, keeping the algorithm
  readable; invariants (seam length == H, seam 4/8-connected) are asserted in tests, not
  re-checked every iteration.
- **Grayscale & RGBA inputs**: `io` normalizes to RGB up front so `core` only ever sees
  `(H,W,3)`.

## Testing (TDD, pytest)

- **`test_core.py`** — the heart: hand-built tiny arrays where the min seam is known by
  hand; verify `energy_map` against a hand-computed 3×3; verify `cumulative_energy` /
  `find_vertical_seam` find the provably-optimal seam; property tests (removing a seam
  drops width by exactly 1, height unchanged, seam is connected). A negative-energy mask
  forces the seam through known columns.
- **`test_features.py`** — resize hits exact target dims; transpose path (height) matches
  manual transpose; `remove_object` on a synthetic image with a bright blob erases the
  blob.
- **`test_io.py`** — round-trip load/save; RGBA→RGB; mask parsing; mismatched mask raises.
- **`test_cli.py`** — argument parsing and error exits on bad input.
- **Determinism**: no randomness → exact assertions.

low-detail background:

1. **Hero narrowing GIF** — records each seam removal into an animated GIF of the image
   continuously narrowing with subjects intact (the 3-second pitch).
2. **Side-by-side vs dumb resize** — seam carving vs naive squish/crop at the same target
   size.
3. **Object-removal before/after** — a masked subject vanishing.
4. **Seam overlay** — chosen seams drawn in red for explanation/debugging.

README leads with the GIF, then the comparison, then the erase-a-person trick, and cites
